# Проект DUMP

## О чем проект и мотивация

Инструмент для создания единого Markdown-дампа проекта, который по большей части написан для себя.
Мотивация: Иногда возникает необходимость передать контекст проекта в LLM или например переслать проект с "красивым" форматированием другу. Копировать файлы вручную - долго, а обычные скрипты часть "захлебываются" на больших проектах и бинарных файлах. Этот же инструмент, написанный буквально за вечер решает проблему эффективно и безопасно.

## Что хорошо

- Zero-RAM Footprint: Благодаря итераторам и `std::io::copy`, программа потребляет константный объем памяти независимо от размера проекта.
- Smart Scanning: Крейт `ignore` позволяет автоматически пропускать файлы из `.gitignore`, не захламляя дамп мусором.

На этом плюсы заканчиваются, ибо это простая cli утилита.

## Что плохо

- Concurrency: Сейчас обход идет в один поток. Для эстримально больших монорепозиториев можно в будущем внедрить парралельный обход.
- Lack of Test Coverage: Отсуствие тестов

## Как использовать и установить

### Установка

1. Склонируйте репозиторий.
2. Соберите проект: `cargo build --release`.
3. Переместите бинарный файл в `$PATH` или запускайте через cargo.

### Использование

```sh
dump .

dump ./src -o project_src.md
```
